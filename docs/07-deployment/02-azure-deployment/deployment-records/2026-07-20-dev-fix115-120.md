## 4. FIX-115 資料面套用（Kudu ad-hoc）

Stage 2 的 prompt 存於 DB `prompt_configs`，`db:seed` 刻意不覆寫既有內容（使用者可能自訂），因此**只換映像不會生效**。`update-stage2-prompt.js` 刻意未接入 entrypoint（避免部署時被動觸發），故以顯式執行為準。

Kudu 是獨立 sidecar、看不到 app container 的 `/app`，故比照 FIX-105 / FIX-110：上傳腳本至 `/home` → 安裝 `pg@8.7.3`（Kudu 為 node v14.19.2）→ 執行。

```
[stage2-prompt] before — id=cmo197zi1000bnsxgkyfge7im version=2 hasKnownFormatsVar=false
[stage2-prompt] done — 1 GLOBAL prompt(s) updated (0 = already up to date)
[stage2-prompt] after  — id=cmo197zi1000bnsxgkyfge7im version=4 hasKnownFormatsVar=true
[stage2-prompt] VERIFY_PASS
```

> stderr 僅有 `pg` 的 SSL 模式棄用警告（ExitCode 0），非錯誤。

🔴 **實際套用版本 = FIX-115 + FIX-121**：腳本取自工作樹，而 FIX-121（15:18:38 提交，同一工作樹的併發 session）也修改了 `update-stage2-prompt.js`。因此寫入的 prompt 含 FIX-121 的「identificationRules 可變部位 inline 標註」。`version` 落在 **4** 而非 FIX-115 本地驗證時的 3，即為此故。該內容已於 16:25 隨 PR #122 併入 `origin/main`，非未審查版本，但範圍確實大於原訂的「僅 FIX-115」。

**預期效果**：Stage 2 可看見已知格式清單 → `matchedKnownFormat` 不再恆為 null → `resolveFormatId` 能命中 → JIT 觸發率下降。連帶 `isNewFormat` 由 true 轉 false，綜合信心度 **+3 分**（`FORMAT_MATCHING` 維度 70→90、權重 15%），本地量測約 6% 的 QUICK_REVIEW 文件會轉為 AUTO_APPROVE。

## 5. 建置失敗與修復（FIX-122）

第一次建置 `ck15` 失敗於型別檢查：

```
./prisma/seed-prod-reference.ts:51:37
Type error: Cannot find module './seed-data/prompt-configs'
```

根因：FIX-118 改用**靜態 import**，但 `.dockerignore:151` 排除整個 `prisma/seed-data`，而 `tsconfig.json` 的 `include` 涵蓋 `prisma/**/*.ts` → builder 階段缺模組。本地 `type-check` 與 CI 皆通過（檔案樹完整），**只有 Docker 分層複製才會顯現**。詳見 [FIX-122](../../../../claudedocs/4-changes/bug-fixes/FIX-122-dockerignore-excludes-prompt-configs-breaks-build.md)。

### 診斷過程的可複用要點

| 問題 | 解法 |
|------|------|
| `az acr build` 本機串流在 `✔` 以 cp1252 崩潰，看不到失敗原因（`PYTHONUTF8` / `PYTHONIOENCODING` / PowerShell 皆無效） | 改用 REST 取 log SAS URL 再 `curl` 下載原始檔：`az rest --method post --url ".../runs/<runId>/listLogSasUrl?api-version=2019-06-01-preview" --query logLink -o tsv`。這是目前唯一能取得**完整**建置日誌的方式 |
| 後續建置避免同一崩潰 | `az acr build --no-logs`（本機不串流，改用 `az acr task show-run` 輪詢狀態） |
| app / SCM 主機名解析不到 | 實際 host 帶唯一後綴 `...-f8dua6b5eqerbrbk.eastasia-01.azurewebsites.net`（`az webapp show --query defaultHostName` 取得）；SCM 走 `nslookup <scm> 8.8.8.8` → `13.75.34.162` + `curl --resolve` |
| Kudu VFS 路徑 | VFS 根即 `/home`，故 `/api/vfs/ops-fix115/x.js` → `/home/ops-fix115/x.js`（寫成 `/api/vfs/home/...` 會落到 `/home/home/...`，回 201 但檔案不在預期位置） |
